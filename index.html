Concepto de Redes Informáticas

Una red informática es un conjunto de computadoras y dispositivos conectados entre sí para compartir información, recursos y servicios.

Permite intercambiar datos, acceder a internet, compartir archivos e impresoras y trabajar colaborativamente.

Clasificación de Redes Informáticas

Según su alcance

  • PAN: Red personal.
  • LAN: Red local.
  • MAN: Red de ciudad.
  • WAN: Red global (Internet).

Según el tipo de conexión

  • Cableada: usa cables o fibra óptica.
  • Inalámbrica: usa WiFi o Bluetooth.

Según su titularidad

Privadas: requieren usuario y contraseña.

Públicas: acceso libre.

Según su tecnología

P2P: sin servidor central.

Cliente-Servidor: con servidor central.

Topologías de Red

Definen cómo se organizan y conectan los dispositivos dentro de una red.

Topología en Estrella

Todos los equipos se conectan a un nodo central.

Ventajas

  • Fácil instalación.
  • No afecta toda la red si falla un cable.

Desventajas

  • Depende del nodo central.
  • Más cableado.

Topología en Bus

Todos comparten un cable principal.

Ventajas

  • Bajo costo.
  • Simple.

Desventajas

  • Si falla el cable, falla todo.

Topología en Anillo

Los equipos forman un círculo.

Ventajas

  • Orden en transmisión.

Desventajas

  • Difícil de ampliar.

Topología en Malla

Todos los dispositivos están conectados entre sí.

Ventajas

  • Muy confiable.

Desventajas

  • Costosa.

Software Cisco Packet Tracer

El software Cisco Packet Tracer es una herramienta de simulación de redes diseñada principalmente para aprender, practicar y experimentar con redes de computadoras sin necesidad de tener los equipos físicos. Es muy usado por estudiantes y profesionales que estudian redes, especialmente aquellos que siguen cursos de Cisco Networking Academy.

Funciones principales

Simulación de redes:

  • Permite crear redes virtuales usando routers, switches, computadoras, servidores, firewalls, y otros dispositivos de red.
  • Puedes conectar dispositivos con cables simulados (Ethernet, fibra, inalámbrico).

Práctica de configuración:

  • Se puede configurar dispositivos usando interfaz de línea de comandos (CLI), como si fuera un equipo real de Cisco.
  • Permite practicar comandos de Cisco IOS, configuraciones de IP, VLANs, protocolos de enrutamiento (como OSPF, RIP, EIGRP), NAT, DHCP, y más.

Visualización del tráfico de red:

  • Muestra cómo viajan los datos dentro de la red usando paquetes simulados.
  • Puedes ver rutas de paquetes, errores de conexión y cómo los dispositivos procesan la información.

Aprendizaje interactivo:

  • Incluye escenarios educativos donde los estudiantes deben resolver problemas de redes, como asignación de IP, configuración de redes inalámbricas o conexión entre subredes.

Pruebas y experimentos sin riesgo:

  • Puedes romper la red, cambiar configuraciones y experimentar sin dañar equipos reales ni afectar una red real.

En resumen, Cisco Packet Tracer es un laboratorio virtual de redes: simula hardware y software de Cisco para que puedas aprender y practicar sin necesidad de comprar routers o switches reales.

Direcciones IP y Redes

1. ¿Qué es una dirección IP?

Una dirección IP (Internet Protocol) es un identificador único asignado a cada dispositivo conectado a una red que utiliza el protocolo de Internet, como computadoras, celulares o routers.

Es como la dirección de tu casa, pero en Internet, para que los datos sepan a dónde ir.

2. Tipos de direcciones IP

a) Según su versión

  • IPv4: 4 números separados por puntos (0–255). Ejemplo: 192.168.0.1. Límite: ~4.3 mil millones de direcciones posibles.
  • IPv6: 8 bloques de 4 caracteres hexadecimales separados por dos puntos. Ejemplo: 2001:0db8:85a3:0000:0000:8a2e:0370:7334. Necesaria por la escasez de IPv4 y permite muchas más direcciones.

b) Según su uso

  • IP Pública: Visible en Internet. Asignada por tu proveedor de Internet (ISP). Ejemplo: 186.130.42.15.
  • IP Privada: Solo funciona dentro de redes locales (hogar, empresa). Ejemplos de rangos:
    • 192.168.0.0 – 192.168.255.255
    • 10.0.0.0 – 10.255.255.255
    • 172.16.0.0 – 172.31.255.255
  • IP Estática vs Dinámica:
    • Estática: No cambia; ideal para servidores o cámaras.
    • Dinámica: Cambia con el tiempo; usada por la mayoría de usuarios domésticos.

3. Cómo se usa una dirección IP

  • Para enviar datos: Permite que los servidores sepan a dónde enviar la información.
  • Para identificar dispositivos: El router asigna IPs privadas a cada dispositivo en casa.

4. Cómo ver tu IP

  • IP pública: Busca "cuál es mi IP" en Google.
  • IP privada:
    • Windows: cmd → ipconfig
    • Mac/Linux: ifconfig o ip addr

5. Clases de IP (IPv4)

Clase Rango del primer octeto Uso principal
A 0 – 127 Redes muy grandes
B 128 – 191 Redes medianas
C 192 – 223 Redes pequeñas / domésticas
D 224 – 239 Multicast (envío a varios destinos)
E 240 – 255 Experimental / investigación

Nota: La clase 127 se reserva para localhost (127.0.0.1), que es tu propio equipo.

6. Cómo saber la clase de tu IP

Toma el primer número de tu IP. Ejemplo:

IP = 192.168.10.5 → primer número = 192 → Clase C

7. ¿Qué es una máscara de subred?

Una máscara de subred define qué parte de la IP corresponde a la red y qué parte a los hosts. Se combina con la IP para determinar si dos dispositivos están en la misma red.

Ejemplo:

  • IP: 192.168.1.10
  • Máscara: 255.255.255.0
  • 255 → parte de red
  • 0 → parte de host

a) Notación

  • Decimal con puntos: 255.255.255.0
  • CIDR: /n indica el número de bits para la red. Ejemplo: 192.168.1.10/24 → equivale a 255.255.255.0

b) Máscaras de subred comunes

Clase IP Máscara típica CIDR Hosts posibles
A 255.0.0.0 /8 16,777,214
B 255.255.0.0 /16 65,534
C 255.255.255.0 /24 254

Cálculo de hosts: 2^n − 2, donde n es el número de bits para hosts (se restan 2 para la dirección de red y broadcast).

8. ¿Qué es la puerta de enlace (gateway)?

La puerta de enlace es un dispositivo (normalmente un router) que conecta tu red local con otras redes, generalmente Internet.

Es como la puerta de tu casa: para salir a Internet, necesitas pasar por ella. Los dispositivos en una red local envían al gateway los datos destinados a otras redes.

a) Relación con IP y máscara

  • IP: identifica al dispositivo.
  • Máscara de subred: define la red local.
  • Puerta de enlace: indica a dónde enviar datos fuera de la red local.

Ejemplo de red doméstica:

  • IP de tu PC: 192.168.1.10
  • Máscara: 255.255.255.0
  • Puerta de enlace: 192.168.1.1 (el router)

Esto significa que cualquier IP fuera del rango 192.168.1.x se enviará al router para su direccionamiento correcto.
